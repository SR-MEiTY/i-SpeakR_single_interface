Modules

Data processing module
Listing out wav files This part will have two options:
Infer:This is for Novices. Users need to provide path to three separate folders. All wav files in each set must be directly placed in the respective folders without and sub-folders.
DEV – Development set files
ENR – Enrollment set files
TEST – Testing set files
Specify: This option for advanced users. User needs to provide path containing csv files. The csv filename for development should start with “DEV”, csv filename for enrollment should start with “ENR”, and the csv filename for testing should start with “TEST”. The csv files need to have the following fields:
utterance_id: Unique ID for each utterances listed in the csv file (required)
speaker_id: Speaker ID corresponding to each utterance in the csv file (required)
wav_path: Path to the wav file for each utterance in the csv file (required)
gender: Gender of the speaker for each utterance (optional)
sensor: Sensor used to record each utterance (optional)
language: Language used for each utterance (optional)
Checking sampling frequency: By default 16000Hz is used as the sampling frequency. Users can specify a different sampling frequency in the config.ini file.
Normalization: By default, the wav files are normalized by mean subtraction and then dividing by the maximum value. Users can add other normalization strategies.
Sub-utterance computation – Segmentation of the wav files to sub-utterance is done. Development and enrollment utterances are chopped into approximately 50s segments. Test utterances are chopped into approximately [10s, 20s, 30s, 40s, 50s, 60s] segments. Care is taken to split the utterances only at silence regions using sample-wise VAD decisions. Voiced samples are selected based on sample energies greater than a threshold of 60% of average energy of the utterance.
Feature Computation: For each sub-utterance a feature is computed.
MFCC: A flag is provided for user to choose whether c0 is required or not. If c0 is not required, then N_MFCC+1 coefficients are computed and c0 is ignored.
VAD: The computed features per frame are selected based on a VAD on the frame-wise energy. The selected frames are stored as a feature file in the output folder.
